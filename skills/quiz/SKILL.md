# Quiz - コード変更内容についての復習

git diff でブランチの変更コードを確認し、ユーザーがコードを1行1行理解しているかチェックするクイズを行う。

## パフォーマンス重要ルール

- **git diff のフル取得は最初の1回だけ。** セッション中は結果を再利用し、再度 git diff を実行しない。
- **git コマンドはまとめて1回の Bash 呼び出しで実行する。**
- **ログはセッション終了時にまとめて1回だけ書く。** 各 Q&A ごとには書かない。

## 手順

### 初回セットアップ（1回だけ）

1つの Bash 呼び出しで以下をまとめて実行する：

```bash
git branch --show-current && git log --oneline -5 && git diff --name-only main...HEAD
```

変更ファイル一覧を取得したら、内部的にリストとして保持する。
diff がない場合は「変更されたコードが見つかりません」と教えて終了する。

### 各質問サイクル

1. 保持している変更ファイルリストからランダムで1ファイル選び、`git diff main...HEAD -- <ファイルパス>` でそのファイルだけの diff を取得する（初回のみ。同じファイルが再度選ばれたら結果を再利用する）
2. diff の中からランダムで1箇所を選ぶ
3. その箇所のコードを引用して「このコードは何をしていますか？」と日本語で質問する
4. 私が答えたら正誤・補足を教える
5. Q&A の内容をメモリ上に保持する（まだファイルには書かない）
6. 「次の質問に進みますか？」とAskUserQuestionツールを使って聞く

質問は具体的なコード片（メソッド名、CSS property、関数など）を引用してください。

### セッション終了時

ユーザーが「終了」「やめる」「No」などを選んだら、保持していた全 Q&A をまとめてログファイルに書き込む。

## ログの記録

全 Q&A のやりとりをセッション終了時に `~/.claude/quiz-logs/YYYY-MM-DD.md`（その日の日付）にまとめて書き込む。

- ディレクトリが存在しなければ `mkdir -p ~/.claude/quiz-logs` で作成する
- ファイルが存在しなければ `Write` で新規作成、存在すれば `Edit` で末尾に追記する
- 1 セッションの最初の Q&A の前に `# YYYY-MM-DD のクイズ` という見出しを書く（既存ファイルなら不要）
- 各 Q&A は次のフォーマットで記録する：

```markdown
## HH:MM (branch: <現在のブランチ名>)

**Q:** （質問文）
\`\`\`<lang>
（引用したコード片）
\`\`\`

**A (私):** （ユーザーの回答をそのまま）

**評価:** （正誤・補足）
```

- 私の回答が「わからない」「スキップ」などの場合もそのまま記録する
